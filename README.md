# Hackatsuon Starter 🐟

気仙沼発ハッカソン **Hackatsuon** のワークショップ用スターターキットです。
ブラウザだけで、AIと一緒にアプリを作って、世界に公開するところまで行きます。

> 完成しなくて大丈夫。今日のゴールは「自分の手で作って、外に出す」体験をすることと、
> 今日の続きをやれる環境を持ち帰ることです。困ったら Discord で聞いてください。

---

## 全体の流れ

1. このテンプレートから自分のリポジトリを作る
2. Codespace(ブラウザ上の開発環境)を起動する
3. AI(Claude Code)を起動する
4. 作りたいものを日本語で伝えて、アプリを作る
5. コードを GitHub に反映する
6. GitHub Pages で世界に公開する

---

## Step 1. 自分のリポジトリを作る

1. このページ右上の緑のボタン **「Use this template」** をクリック
2. **「Create a new repository」** を選択
   (「Open in a codespace」は選ばないでください。作品が保存されない状態になります)
3. Repository name に好きな名前を入力(例: `my-hackatsuon-app`)
4. **「Create repository」** をクリック

これで「自分のプロジェクト」ができました。

## Step 2. Codespace を起動する

1. 自分のリポジトリのページで、緑の **「Code」** ボタンをクリック
2. **「Codespaces」** タブ → **「Create codespace on main」** をクリック
3. ブラウザ上にエディタ(VS Code)が開きます。**このあと裏でセットアップが
   2〜3分動きます。** ターミナルに入力できるように見えても、まだ待ってください

**準備完了の確認**: ターミナルの **`+` ボタンで新しいターミナルを開いて**ください。

- 「🐟 Hackatsuon Starter へようこそ!」が出たら **準備完了**。そのターミナルで作業します
- 出ないときは1〜2分待って、もう一度新しいターミナルを開く

> ターミナルが見えないときは、メニュー(≡) → Terminal → New Terminal

## Step 3. AI(Claude Code)を起動する

配布された **APIキー** を使います。🐟が出たターミナルに以下を入力:

```bash
export ANTHROPIC_API_KEY=ここに配布されたキーを貼り付け
```

続けて、Claude Code を起動:

```bash
claude
```

初回は質問が3回ほど出ます。**すべて「1」が選ばれたまま Enter** でOKです。

1. 「Do you want to use this API key?」→ **1. Yes**
   ⚠️ 「2. No **(recommended)**」と表示されますが、釣られないでください。
   今日は配布されたキーを使うので **Yes が正解** です
2. 「Use Claude Code's terminal setup?」→ 1. Yes
3. 「Is this a project you created or one you trust?」→ 1. Yes, I trust this folder
   (Claude Code にこのフォルダ内でのファイル編集・コマンド実行を許可する、という意味です)

## Step 4. アプリを作る

Claude Code に、作りたいものを **日本語で** 伝えてください。例:

```
index.html を書き換えて、気仙沼の観光スポットを紹介する
ページを作ってください。スマホでも見やすくしてください。
```

```
潮の満ち引きの時間をメモできる簡単なアプリにしてください。
データはブラウザに保存されれば十分です。
```

途中で動作を見たいときは、**別のターミナル**(+ボタンで追加)で:

```bash
http-server -p 8080
```

を実行すると、プレビューが開きます。

**コツ**
- 一度に全部作ろうとせず、少しずつ注文する
- 気に入らなかったら「もっと〇〇にして」と遠慮なく言う
- エラーが出たら、エラーメッセージをそのままAIに貼り付ける
- `/cost` と打つと、ここまでのAI利用料金が見られます

## Step 5. コードを GitHub に反映する

いま作ったものは、まだこの Codespace の中にしかありません。
GitHub に push して、あなたの作品として残しましょう。ターミナルで:

```bash
git add -A
git commit -m "アプリ作成"
git push
```

> コミットメッセージ(-m のあと)は自由です。日本語でOK。

push できたら、自分のリポジトリのページをブラウザで開いてみてください。
あなたのコードが公開されています。世界中の誰でもこのコードを読んで、
学んで、「こうしたらもっと良くなるよ」と提案できる状態です。
午前に話した、あの世界の一員になった、ということです。

これ以降コードは GitHub に残ります。
Codespace を消しても作品は消えません。続きはいつでもやれます。

## Step 6. GitHub Pages で世界に公開する

コードが GitHub にあるなら、公開まではあと数クリックです。

1. 自分のリポジトリのページで **Settings** タブを開く
2. 左メニューの **Pages** をクリック
3. 「Build and deployment」の Source が **Deploy from a branch** になっていることを確認
4. Branch を **main** / **(root)** にして **Save**
5. **1〜2分待つ**(ここが我慢どころ。コーヒーでも)
6. ページ上部に表示される URL を開く:
   `https://あなたのユーザー名.github.io/リポジトリ名/`

**それがあなたのアプリの、世界に公開されたアドレスです。** スマホで開いてみてください 🎉

これ以降は、**`git push` するたびに自動で公開ページも更新**されます(反映まで1〜2分)。

## 今日のあとも続けるには

- 作業場(Codespace)は https://github.com/codespaces からいつでも再開できます
  (無料枠: 月120コア時間。使い終わったら **Stop** しておくと節約になります)
- 再開したら、Step 3 の `export ANTHROPIC_API_KEY=...` からやり直せば続きができます
- ワークショップのAPIキーには有効期限があります。切れたあとの選択肢は Discord で案内します
- データ保存(DB)がほしい、ログイン機能がほしい、など次の一歩も Discord で聞いてください。
  段階的なガイドを用意していきます

## コラム: 今日の道具は、ぜんぶ入れ替え可能

今日は GitHub にいろいろ集めました。コードの置き場も、開発環境(Codespaces)も、
公開(Pages)も。「分散が大事って言ってたのに、1社に集めてるじゃないか」——
その指摘は正しいです。だから、種明かしをしておきます。

**Git そのものは分散設計です。** あなたの Codespace にある clone は、履歴ごと
完全な複製。かりに GitHub が明日消えても、コードは全員の手元に残ります。
GitHub は「便利な集会所」であって、「所有者」ではありません。

そして、今日使った道具にはすべて代わりがあります:

| 今日使ったもの | 代わりの例 |
|---|---|
| GitHub (コード置き場) | GitLab、Codeberg、自分のサーバに Forgejo を立てる |
| GitHub Pages (公開) | Cloudflare Pages、Netlify、Vercel、自前サーバ |
| Codespaces (開発環境) | 手元の PC + VS Code、WSL、好きなエディタ |
| Claude Code (AI) | Codex CLI、Gemini CLI、手元のPCで動くオープンなモデル |

今日の組み合わせは「初日にいちばんつまずきが少ない一例」にすぎません。
**選択肢があると知っていて、必要なら選び直せること** — それが、今日話した
「技術的自立」の中身です。
